**Canonical entry point:** `match.py` (chunked + checkpointed; outputs `auto_matched.jsonl`, `early_filtered.jsonl`, `failed_entity_resolution.jsonl`, `df_fast.csv`, `unmatched_fast.xlsx`). Formerly `match_fast.py`; the previous `match.py` was archived to `resolve_archive/match_old.py`.

Before entity res, candidates are generated by: 

**Fetching candidates from NPI**
- Query A: First name 2-char prefix match + exact last name match
- Query B: Exact first name match + last name 2-char prefix match

**Geographic filtering**
- If source agency is known, resolve to county via agency lookup and filter candidates to only those with any employment history in that county.

**Date filtering**
- Employment dates must overlap incident year with ±1 year buffer: `post_start_year ≤ incident_year + 1 AND post_end_year ≥ incident_year - 1`

**Candidates returned**
- To be considered, candidates must fall into one of two criteria:
  - (First name 2-char prefix match + full last name match + date_in_range) OR
  - (Full first name match + last name 2-char prefix match + date_in_range) 

# Manual Review

**Skip review IF ALL true**
- first_name exact or fuzzy match 
- last_name exact match 
- source_agency matches or fuzzy-matches post_agency_name


**Review IF ANY true**
Agency mismatch:
   - source_agency != post_agency_name, except for cases where post_agency_name is in "extracted_mentioned_agencies" 
   - mention agency is non-LE (DA, Coroner, ME, etc.), except for cases where post_agency_name is in "extracted_mentioned_agencies"

> **Implementation note (agency validation).** `helpers.validate_agency_match` runs two checks: (1) a deterministic **non-LE guard** that rejects when every agency to compare against is non-LE (DA/Coroner/ME/Public Defender/AG) and the POST agency is LE (Police/Sheriff/Marshal/Patrol/etc.) — this is bypassed when any LE agency appears in `mentioned_agencies`; (2) if the guard does not fire, an **LLM** decides whether the POST agency matches any source/mentioned agency. The LLM now uses the **standard OpenAI API** with model `gpt-5.4-nano` (`llm.py`); the previous Azure endpoint was decommissioned. **LLM response caching is disabled** — every validation is a fresh call.

   - Multiple candidates scored > 0.5 for same mention / multiple candidates with the same name worked at the same agency within the same time period

## flags

Check 1 (Common name ambiguity): Flags when the mention has a common last name AND there are 2+ different officers with the exact same full name (first name and last name) who worked at relevant agencies during the relevant time period.

Check 2 (Same name, different persons, which is redundant with the above but that's fine): Flags when there are 2+ different officers with the exact same full name who worked at relevant agencies during the relevant time period. 

Check 3 (Multiple plausible persons): Flags when there are 2+ different officers with the exact last name, high match probability (>0.75), AND who worked at a matching agency during the relevant time period.
