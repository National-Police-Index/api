Before entity res, candidates are generated by: 

**Fetching candidates from NPI**
- Query A: First name 2-char prefix match + exact last name match
- Query B: Exact first name match + last name 2-char prefix match

**Geographic filtering**
- If source agency is known, resolve to county via agency lookup and filter candidates to only those with any employment history in that county.

**Date filtering**
- Employment dates must overlap incident year with ±1 year buffer: `post_start_year ≤ incident_year + 1 AND post_end_year ≥ incident_year - 1`

**Candidates returned**
- To be considered, candidates must fall into one of two criteria:
  - (First name 2-char prefix match + full last name match + date_in_range) OR
  - (Full first name match + last name 2-char prefix match + date_in_range) 

# Manual Review

**Skip review IF ALL true**
- first_name exact or fuzzy match 
- last_name exact match 
- source_agency matches or fuzzy-matches post_agency_name


**Review IF ANY true**
Agency mismatch:
   - source_agency != post_agency_name, except for cases where post_agency_name is in "extracted_mentioned_agencies" 
   - mention agency is non-LE (DA, Coroner, ME, etc.), except for cases where post_agency_name is in "extracted_mentioned_agencies"

Potential name collisions:
   - Common last names (top 100 CA surnames from https://forebears.io/united-states/california/surnames)
   - Multiple candidates scored > 0.5 for same mention / multiple candidates with the same name worked at the same agency within the same time period 

**Candidates that were not assigned a match**
- Ignore for now?
