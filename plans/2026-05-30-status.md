- **Decision needed (can't fix in code alone):** either (a) point `AZURE_ENDPOINT` at a live Azure OpenAI deployment, or (b) repoint `llm.py` to standard OpenAI ‚Äî `api.openai.com` IS reachable and `OPENAI_API_KEY` is already in `.env` (would swap `AzureOpenAI(...)` ‚Üí `OpenAI(...)` and drop the azure_endpoint/api_version args).
- Hardening already applied this session (kept): validation concurrency `100 ‚Üí VALIDATION_MAX_WORKERS=8` (`match_fast.py`), retry now `wait_random_exponential` w/ jitter, 6 attempts (`llm.py`). Good practice, but orthogonal to the outage.
